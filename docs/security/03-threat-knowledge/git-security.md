## 特に危険な Git コマンド

| コマンド | 何が起きる？ | なぜ危険？ |
|----------|-------------|----------|
| `git push --force` | リモートリポジトリの履歴を強制上書き | 他のメンバーのコミットが消える可能性がある |
| `git push -f` | 上と同じ（短縮形） | 同上 |
| `git reset --hard` | ローカルの変更をすべて破棄して特定の時点に戻る | 未コミットの作業が全て消える |
| `git clean -fd` | Gitで管理されていない全ファイルを強制削除 | 新しく作ったファイルが全て消える |

## 安全な Git の使い方

### 1. コミット前に必ず変更を確認

```bash
git diff          # 変更内容を確認
git diff --staged # ステージ済みの変更を確認
```

### 2. 強制プッシュの代わりに安全な方法を使う

```bash
# 危険: 他の人の変更を上書きする可能性
git push --force

# 安全: 他の人の変更があった場合はエラーになる（安全弁付き）
git push --force-with-lease
```

### 3. ハードリセットの代わりに一時退避を使う

```bash
# 危険: 変更が完全に消える
git reset --hard

# 安全: 変更を一時退避して後から戻せる
git stash
```

### 4. ブランチを作ってから作業する

```bash
# main ブランチを直接変更しない
git checkout -b feature/my-work
```
