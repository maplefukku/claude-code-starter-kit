# MCP サーバーとサブエージェントの安全管理

## MCP サーバーとは？

MCP（Model Context Protocol）サーバーは、Claude Code に追加機能を提供する外部プログラムです。

> **たとえるなら:** スマートフォンのアプリのようなもの。便利な機能を追加できますが、悪意のあるアプリを入れると情報が漏洩するリスクがあります。

## MCP に関するリスク

| リスク | 説明 |
|--------|------|
| 未承認の MCP サーバーの追加 | 悪意のあるサーバーが追加され、データが外部に送信される |
| MCP 経由のデータ漏洩 | MCP ツールを通じて機密情報が外部サービスに送られる |
| MCP サーバーの脆弱性 | MCP サーバー自体にセキュリティ上の欠陥がある |

## 安全に使うために

1. **必要な MCP サーバーだけを有効にする** — 使っていないものは削除
2. **信頼できるソースからのみインストールする** — 公式・信頼できるリポジトリのものを使う
3. **`/mcp` コマンドで定期的に確認する** — 見覚えのないサーバーが追加されていないかチェック
4. **managed-settings で制限する**（チーム向け） — `allowManagedMcpServersOnly: true` で管理者が許可したもの以外を禁止

## サブエージェントとは？

サブエージェントは、Claude Code がタスクを分割して別のAIに委任する仕組みです。

> **たとえるなら:** あなたの代わりに作業してくれるアシスタント（Claude Code）が、さらに別のアシスタントに一部の仕事を頼むようなものです。

### サブエージェントの注意点

- サブエージェントに機密情報を渡さない
- サブエージェントにセキュリティ設定の変更を委任しない
- サブエージェントの出力も信頼できないデータとして扱う
