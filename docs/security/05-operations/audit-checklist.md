# 定期監査のすすめ

[< ガイドトップに戻る](../README.md)

---

セキュリティ設定は、一度設定したら終わりではありません。**週に1回程度**の定期チェックを推奨します。

## チェックコマンド

| コマンド | 確認できること |
|----------|---------------|
| `/permissions` | 現在有効なPermissionsルールの一覧 |
| `/status` | どの階層の設定ファイルが有効か |
| `/hooks` | 登録されているHooksの一覧 |
| `/mcp` | 有効なMCPサーバーの一覧 |
| `claude --version` | Claude Code のバージョン（最新のセキュリティ修正が適用されているか） |

## 週次チェックリスト

- [ ] Permissions ルールが意図通りか？ 不要な `allow` ルールが追加されていないか？
- [ ] Hook Guard が正常に登録されているか？
- [ ] Sandbox が有効になっているか？
- [ ] 見覚えのない MCP サーバーが追加されていないか？
- [ ] `.claude/security-audit.log` にブロックイベントがないか？あれば内容を確認
- [ ] Claude Code のバージョンが最新か？（CVE対策のため v2.1.53以上推奨）
- [ ] `.gitignore` に機密ファイルのパターンが含まれているか？

## 既知の CVE（脆弱性情報）

以下の脆弱性対策のため、**v2.1.53以上**の使用を推奨します。

| CVE ID | 内容 |
|--------|------|
| CVE-2026-25725 | Sandbox escape（サンドボックスからの脱出） |
| CVE-2026-25724 | Symlink bypass（シンボリックリンクによるアクセス制限の迂回） |
| CVE-2026-33068 | Workspace trust bypass（ワークスペース信頼設定の迂回） |

```bash
# バージョン確認
claude --version
```
