# Permissions（許可ルール）— 何を禁止し、何を許可するか

[< ガイドトップに戻る](../README.md)

---

## 基本の考え方：Deny-First（まず全部禁止）

本ツールキットでは **「まず禁止、必要なものだけ許可」** という方針をとっています。これは一般的なセキュリティのベストプラクティスです。

> **たとえるなら:** 「来客は全員受付を通してください。事前に名前が登録されている人だけ通します」という方式です。

## 設定ファイルの場所

```
.claude/settings.json
```

## 禁止ルールの構成

設定ファイルの `permissions.deny` に、禁止する操作がリストアップされています。本ツールキットでは **77個** の禁止ルールが用意されています。

### ルールの読み方

```json
"Bash(rm -rf *)"
```

これは「`rm -rf *`（ファイルの一括削除）に一致するコマンドの実行を禁止する」という意味です。

```json
"Read(**/.env*)"
```

これは「`.env` ファイル（パスワードやAPIキーが入った設定ファイル）の読み取りを禁止する」という意味です。

## 禁止ルールのカテゴリ一覧

| カテゴリ | ルール数 | 説明 |
|----------|----------|------|
| ファイル削除 | 4 | `rm -rf` 系の破壊的な削除 |
| リモートスクリプト実行 | 4 | `curl \| bash` 等、ネットから取ってきたスクリプトの即時実行 |
| 管理者権限 | 1 | `sudo`（管理者としてコマンドを実行）の使用禁止 |
| Git 操作 | 4 | 強制プッシュ、ハードリセットなど |
| シェルバイパス | 3 | `sh -c`, `bash -c` 等、制限を迂回する方法 |
| インフラ破壊 | 5 | `terraform destroy`, `kubectl delete` 等 |
| ネットワーク | 4 | `nc`（ネットキャット）リスナー、簡易HTTPサーバー |
| エンコーディング迂回 | 2 | `base64 -d`（データの復号化で悪意あるコードを隠す手法） |
| その他の危険操作 | 4 | `chmod 777`（全員に全権限）, `eval`（文字列をコードとして実行）等 |
| 機密ファイル読み取り | 13 | `.env`, `.ssh`, `.aws`, 秘密鍵等 |
| 機密ファイル編集 | 6 | 同上ファイルの編集禁止 |
| 機密ファイル書き込み | 6 | 同上ファイルの新規作成禁止 |
| 機密ファイル表示 | 16 | `cat`, `head`, `tail` 等での閲覧禁止 |

## 許可ルールの追加方法

特定の操作を許可したい場合は、`permissions.allow` に追加します。

```json
{
  "permissions": {
    "deny": [ ... ],
    "allow": [
      "Bash(npm test)",
      "Bash(npm run build)"
    ]
  }
}
```

> **注意:** 許可ルールは慎重に追加してください。1つの穴がセキュリティ全体を弱める可能性があります。

## Bypass モードの無効化

Claude Code には、Permissionsルールを一時的にスキップする「Bypass モード」があります。セキュリティを強化するには、これを無効にします。

```json
{
  "permissions": {
    "disableBypassPermissionsMode": "disable"
  }
}
```
