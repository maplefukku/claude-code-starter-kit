# 防御アーキテクチャ — 多層防御の全体像

[< ガイドトップに戻る](../README.md)

---

本ツールキットは、**多層防御（Defense in Depth）** という考え方に基づいて設計されています。これは「1つの防御が突破されても、次の層が守ってくれる」という仕組みです。

> **たとえるなら:** 家のセキュリティで「鍵」「防犯カメラ」「警備会社」を組み合わせるのと同じ考え方です。鍵が壊されても、カメラが記録し、警備会社が駆けつけます。

## 防御の流れ

```
┌─────────────────────────────────────────────────┐
│              あなたの入力（プロンプト）               │
│                       ↓                          │
│  ┌───────────────────────────────────────────┐   │
│  │  第1層: プロンプトインジェクション検出        │   │
│  │  （不正な指示パターンを自動検出してブロック）    │   │
│  └───────────────────┬───────────────────────┘   │
│                       ↓                          │
│  ┌───────────────────────────────────────────┐   │
│  │  第2層: Permissions + PreToolUse           │   │
│  │  （危険なコマンド・ファイルアクセスをブロック）   │   │
│  └───────────────────┬───────────────────────┘   │
│                       ↓                          │
│  ┌───────────────────────────────────────────┐   │
│  │  第3層: Sandbox（OS レベルの隔離）           │   │
│  │  （アクセスできるフォルダ・通信先を制限）       │   │
│  └───────────────────┬───────────────────────┘   │
│                       ↓                          │
│  ┌───────────────────────────────────────────┐   │
│  │  第4層: PostToolUse 出力監視                │   │
│  │  （実行結果に含まれる不正パターンを警告）       │   │
│  └───────────────────────────────────────────┘   │
│                                                   │
│  + CLAUDE.md ポリシー（AI自身の行動規範）           │
│  + .gitignore（機密ファイルの Git 除外）            │
│  + 監査ログ（すべてのブロック記録）                  │
└─────────────────────────────────────────────────┘
```

## 設計原則

| 原則 | 説明 |
|------|------|
| **Deny-First（まず禁止）** | 「まず全部禁止、必要なものだけ許可」の方針 |
| **多層防御** | 1つの防御が突破されても次の層が守る |
| **監査可能** | すべてのセキュリティイベントをログに記録 |
| **モジュラー** | 必要な機能だけを個別に選択できる |
| **非破壊** | 既存設定をバックアップし、マージで追加 |
